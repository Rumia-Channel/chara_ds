# Function tool / Actor Guard review findings (2026-04-29)

Issues found before fixes:

1. `persona_seed.norm_profile_ids` and `persona_seed.explicit_overrides_from_user_txt` are generated by Persona Controller but are not passed to Actor Guard. Guard therefore cannot distinguish explicit source overrides from general age/gender norm defaults.

2. `call_deepseek_tool()` uses `tool_choice: auto` and has fallback paths that parse `message.content` or `reasoning_content` as JSON. Those fallback paths do not receive server-side strict tool schema validation, while local validators are currently weak.

3. `validate_persona_output()` only checks that `persona_seed` is a dict. It does not enforce required schema fields when fallback parsing bypasses tool strict validation.

4. `validate_actor_guard_output()` does not require `suggested_fix_ja`, even though it is passed back to Actor for regeneration.

5. `scenario_constraints.medium` enum is narrower than the expanded `global_style.genre` and lacks common scene categories such as school/home/outdoor/horror/psychological conflict/other.

6. Documentation is stale for expanded `age_band` values and age_gender_norms examples.

Fix direction:
- Pass persona-level norm metadata into Actor Guard payload.
- Teach actor_guard prompts to use `norm_profile_ids` and explicit overrides.
- Add local schema validation for fallback parsed tool outputs, at least for persona and guard required fields.
- Expand `scenario_constraints.medium` enum and update persona prompts/docs accordingly.
